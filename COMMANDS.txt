OPTION 1 — STREAMLIT WEB APP ONLY (easiest, no phone/backend)
================================================================
  This runs the full dashboard in your browser. No iOS, no API, no ngrok.
  Streamlit reads cloudscout.db directly.

  Without AI Scout chat:
     cd /Users/lol/Documents/Repos/cloud-scout
     venv/bin/python3 -m streamlit run app.py

  With AI Scout chat enabled:
     cd /Users/lol/Documents/Repos/cloud-scout
     ANTHROPIC_API_KEY=sk-ant-... venv/bin/python3 -m streamlit run app.py

  → Browser opens at http://localhost:8501 automatically.
  → Press Ctrl+C in the terminal to stop.


================================================================
OPTION 2 — iOS APP, SAME WIFI  (Mac + iPhone on same network)
================================================================
  Easiest iOS path. No ngrok needed.

  Step 1 — Find your Mac's local IP:
     ipconfig getifaddr en0
     → e.g. 192.168.1.42   (use en1 if you're on Ethernet/Thunderbolt)

  Step 2 — Start the FastAPI backend, bound to your LAN:
     cd /Users/lol/Documents/Repos/cloud-scout
     venv/bin/uvicorn api:app --reload --host 0.0.0.0 --port 8000

     (For AI endpoints, prefix with ANTHROPIC_API_KEY=sk-ant-...)

  Step 3 — Point the iOS app at your Mac.
     Open ios/CloudScout/APIClient.swift, line 5. Replace the URL
     in quotes after `??` with your Mac's IP. Leave the env-var
     key in brackets alone:

        let API_BASE = ProcessInfo.processInfo.environment["API_BASE"]
                       ?? "http://192.168.1.42:8000"

  Step 4 — Build and run:
     open ios/CloudScout.xcodeproj
     → Pick your iPhone (or a simulator) from the device menu in Xcode
     → Press Cmd+R

  Notes:
    - Simulator can use http://127.0.0.1:8000 instead — and you can drop
      `--host 0.0.0.0` from Step 2 in that case.
    - If the IP stops working later, your router reassigned it. Rerun
      ipconfig and update APIClient.swift.


================================================================
OPTION 3 — iOS APP, DIFFERENT WIFI  (phone on cell or another network)
================================================================
  Use this when your iPhone can't reach your Mac's LAN. Requires ngrok.

  Step 1 — Start the backend (default localhost is fine):
     cd /Users/lol/Documents/Repos/cloud-scout
     venv/bin/uvicorn api:app --reload

     (For AI endpoints, prefix with ANTHROPIC_API_KEY=sk-ant-...)

  Step 2 — Open a public tunnel in a SECOND terminal:
     ngrok http 8000
     → Copy the https://xxxx.ngrok-free.app URL ngrok prints.

  Step 3 — Point the iOS app at the tunnel.
     Open ios/CloudScout/APIClient.swift, line 5. Replace the URL
     in quotes after `??` with the ngrok URL:

        let API_BASE = ProcessInfo.processInfo.environment["API_BASE"]
                       ?? "https://xxxx.ngrok-free.app"

  Step 4 — Build and run:
     open ios/CloudScout.xcodeproj
     → Pick your iPhone (or a simulator) from the device menu in Xcode
     → Press Cmd+R

  Notes:
    - ngrok free plan gives a NEW URL every restart. Update APIClient.swift
      and rebuild each time you restart ngrok.
    - Keep both terminals (uvicorn + ngrok) running while you use the app.


================================================================
OPTION 4 — EXPO / REACT NATIVE APP  (mobile/ directory)
================================================================
  Run ALL of these from: /Users/lol/Documents/Repos/cloud-scout/mobile

  First time setup (once):
     cd /Users/lol/Documents/Repos/cloud-scout/mobile
     npm install

  ─── Same WiFi ──────────────────────────────────────────────
  Tab 1 — Backend bound to LAN (from repo root):
     cd /Users/lol/Documents/Repos/cloud-scout
     venv/bin/uvicorn api:app --reload --host 0.0.0.0 --port 8000

  Tab 2 — Find your Mac's IP:
     ipconfig getifaddr en0      # e.g. 192.168.1.42

  Tab 3 — Point Expo at your Mac and start it:
     cd /Users/lol/Documents/Repos/cloud-scout/mobile
     echo "EXPO_PUBLIC_API_URL=http://192.168.1.42:8000" > .env
     npx expo start -c
     → Scan the QR code with Expo Go on your phone.

  ─── Different WiFi (ngrok) ─────────────────────────────────
  Tab 1 — Backend:
     cd /Users/lol/Documents/Repos/cloud-scout
     venv/bin/uvicorn api:app --reload

  Tab 2 — ngrok tunnel:
     ngrok http 8000
     → Copy the https://xxxx.ngrok-free.app URL.

  Tab 3 — Point Expo at the tunnel:
     cd /Users/lol/Documents/Repos/cloud-scout/mobile
     echo "EXPO_PUBLIC_API_URL=https://xxxx.ngrok-free.app" > .env
     npx expo start -c

  Notes:
    - EXPO_PUBLIC_* env vars are baked in at start. You MUST restart Expo
      with `-c` after editing .env, or the old URL will still be used.
    - ngrok free URL changes every restart — update mobile/.env and
      restart Expo each time.


================================================================
COMMAND-LINE TOOLS  (analytics over the local cloudscout.db)
================================================================

TEAM STATS (recent form, averages, rolling performance):
  python main.py --team "LA Lakers" --last 5
  python main.py --team "Boston Celtics" --last 10
  python main.py --team "Warriors" --last 3

GAME RESULTS (raw W/L results with scores and location):
  python main.py --games "LA Lakers" --last 5
  python main.py --games "Nuggets" --last 10

PLAYER STATS (average stats over last N games):
  python main.py --player "LeBron James" --last 10
  python main.py --player "Luka Doncic" --last 5
  python main.py --player "Jayson Tatum" --last 15

PLAYER VS TEAM (player's averages against a specific opponent):
  python main.py --pvt "LeBron James" "Chicago Bulls" --last 5
  python main.py --pvt "Luka Doncic" "Celtics" --last 10

HEAD-TO-HEAD (last N meetings between two teams):
  python main.py --h2h "LA Lakers" "Boston Celtics" --last 5
  python main.py --h2h "Warriors" "Nuggets" --last 10

TOP PERFORMERS (rank players on a team by avg points, assists, rebounds, steals):
  python main.py --top "LA Lakers" --last 5
  python main.py --top "Boston Celtics" --last 10

SCRAPE FRESH DATA (fetch new games from NBA.com before analysis):
  python main.py --team "LA Lakers" --last 5 --scrape
  python main.py --games "LA Lakers" --last 10 --scrape
  python main.py --h2h "LA Lakers" "Boston Celtics" --last 5 --scrape
  python main.py --top "LA Lakers" --last 5 --scrape

SCRAPE ALL TEAMS (fetch last N games for all 30 NBA teams):
  python main.py --scrape-all --last 15
  python main.py --scrape-all --last 10

RUN NIGHTLY SCHEDULER (auto-scrapes watchlist teams at midnight):
  python scheduler.py

HELP:
  python main.py --help

Notes:
  - --last defaults to 5 if not specified
  - Team names accept full names, nicknames, or abbreviations
    (e.g., "Los Angeles Lakers", "LA Lakers", "Lakers", "LAL")
  - Player names require full name (e.g., "LeBron James", "Luka Doncic")
  - Use --scrape on the first run to populate the database
  - Data is saved locally in cloudscout.db so you only need to scrape once
  - Add --scrape to any command to fetch fresh data before analysis
